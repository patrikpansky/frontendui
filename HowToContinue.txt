# Systém pro správu přijímacího řízení

## Hlavní funkce:
1. Zobrazení podaných přihlášek
   - Stránka studijního programu
   - Stránka přijímacího řízení
2. Podle studijního programu
3. Stav přihlášek
4. Správa podmínek zkoušek (CRUD operace)
5. Správa termínů zkoušek

## Aktuální implementace:
- Pouze zobrazení (entity: program)
- Ukázka GraphQL dotazu:
```graphql
query MyQuery {
  programPage {
    id
    name
  }
}
```

## Další vývojové úkoly:
- Vygenerovat npm balíček s atributy:
  - Přihlášky
  - Program
  - Zkouška
- Uložení do pack_bravo
## Uživatelské rozhraní:
1. Seznam studijních programů
   → Detail programu
   → Seznam přihlášek
   → Detail přihlášky
2. Seznam zkoušek
   → Úprava termínu zkoušky
## Datový model:
- Hierarchie:
  - Studijní programy → Přihlášky → Přihláška uživatele
  - Správa podmínek zkoušek → Správa termínů
- Demo data: system-data.hk.json
- Struktura:
  - Názvy tabulek = key
  - Hodnoty = vektory (optimální pro ukládání)
## Generování testovacích dat:
- 10 datových bodů pomocí mutací queries
- Příklad workflow:
  - Vložení programu (program insert)
  - Následné vkládání předmětů (subject inserts)
